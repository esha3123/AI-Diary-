## 🎯 PROJECT OVERVIEW
═══════════════════════════════════════════════════════════════

AI-Diary is a sophisticated personal journaling application that combines 
traditional diary writing with cutting-edge AI technology. The platform allows 
users to create, manage, and share diary entries while receiving personalized 
AI insights powered by Google Gemini API.

### Key Innovation:
- 🧠 AI-powered insights in Hinglish (Hindi + English)
- 🎭 Bollywood-style personality responses
- 📊 Mood tracking and analytics
- 🔊 Text-to-Speech with filmy voiceovers
- 🌐 Public/Private entry sharing
- ✨ Beautiful animated user interface

## 🏗️ TECHNICAL ARCHITECTURE
═══════════════════════════════════════════════════════════════

### Backend Framework:
- **Server**: Node.js with Express.js
- **Database**: MongoDB with Mongoose ODM
- **Authentication**: Passport.js with Local Strategy
- **Session Management**: Express-session
- **Template Engine**: EJS with EJS-Mate
- **AI Integration**: Google Gemini 1.5 Flash API

### Frontend Technologies:
- **Styling**: Bootstrap 5 + Custom CSS
- **Animations**: Custom JavaScript with Canvas particles
- **Icons**: Font Awesome 6.5.1
- **Interactions**: Vanilla JavaScript ES6+

### Database Schema:
```
Users Collection:
├── username (String, required, unique)
├── email (String, required, unique)
├── password (hashed via passport-local-mongoose)
├── createdAt (Date, auto-generated)

DiaryEntries Collection:
├── title (String, required)
├── content (String, required)
├── mood (String, enum: ['😂','😀','🤗','😤','😔','😰'])
├── isPrivate (Boolean, default: true)
├── likes (Number, default: 0)
├── likedBy (Array of user IPs)
├── createdAt (Date, default: now)
├── comments (Array, refs to Comment collection)
├── owner (ObjectId, ref to User)

Comments Collection:
├── comment (String, required)
├── count (Number, default: 0)
├── likedBy (Array of user IPs)
├── createdAt (Date, default: now)
```

```bash
# Google Gemini Configuration
GEMINI_API_KEY=your_actual_gemini_api_key_here

# Session Configuration  
SESSION_SECRET=your_secure_session_secret

# Database Configuration
MONGODB_URL=mongodb://127.0.0.1:27017/Dear-diary
MONGO_DB_ATLAS=your_mongodb_atlas_connection_string

# Application Environment
NODE_ENV=production
PORT=5000
```

### Installation & Running:
```bash
# 1. Install dependencies
npm install

# 2. Set up environment variables
cp .env.example .env
# Edit .env with your actual values

# 3. Start MongoDB (local)
mongod

# 4. Run application
npm start          # Production
npm run app        # Development (with nodemon)

# 5. Access application
http://localhost:5000
```
